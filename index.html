PART III

SOCKET.IO

Websockety

  • Zaimplementowane przez większość przeglądarek i webserverów
  • Poprzedzone handshakiem (przegladarki interpretuja go jako HTTP Upgrade)
  • Realtime
  • Pełny duplex
  • WS + WSS

Socket.io

  • JSowa biblioteka
  • Dostarcza uproszczone API do wysyłania i odbierania komunikatów na końcówkach połączenia
  • Można powiedzieć że to jest taki wrapper na websockety
  • Dodaje np. rozgłaszenie komunikatów do wszystkich websocketów / dowiązywanie danych do konkretnego websocketa

API klienta Socket.io

  • .open()
  • .on(eventName, ()=>{});
  • .emit(eventName,[args])
  • .close()
